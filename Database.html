SQL Database Guide

Comprehensive reference for Structured Query Language (SQL)

üìä Introduction to SQL

SQL (Structured Query Language) is used for accessing and manipulating databases.

SQL keywords are NOT case sensitive: select is the same as SELECT.

üéØ Use of SQL

SQL can:

  • Execute queries against a database
  • Retrieve data from a database
  • Insert records in a database
  • Update records in a database
  • Delete records from a database
  • Create new databases and tables
  • Create stored procedures and views
  • Set permissions on tables, procedures, and views

üóÉÔ∏è Database Defined

A database is an organized collection of structured information typically stored electronically in a computer system.

Data is typically modeled in rows and columns in a series of tables for efficient processing and querying.

Most databases use structured query language (SQL) for data management.

üîß SQL Commands

Some of the most important SQL commands:

Command Description
SELECT Extracts data from a database
UPDATE Updates data in a database
DELETE Deletes data from a database
INSERT INTO Inserts new data into a database
CREATE DATABASE Creates a new database
ALTER DATABASE Modifies a database
CREATE TABLE Creates a new table
ALTER TABLE Modifies a table
DROP TABLE Deletes a table
CREATE INDEX Creates an index (search key)
DROP INDEX Deletes an index

üîç SELECT Query

Used to fetch data from database tables.

-- Fetch all data from a table
SELECT * FROM table_name;

-- Fetch specific column
SELECT column_name FROM table_name;

-- Fetch multiple columns
SELECT column1, column2, column3 FROM table_name;

‚ú® DISTINCT

Returns only distinct (different) values.

SELECT DISTINCT column1, column2
FROM table_name;

üìà ORDER BY

Sorts the result-set in ascending or descending order.

SELECT column1, column2
FROM table_name
ORDER BY column1 ASC|DESC;

üîé WHERE Clause

Filters records based on specified conditions.

SELECT column1, column2
FROM table_name
WHERE condition;
Example:
SELECT * FROM Customers WHERE Country='Brazil';

üßÆ AND, OR, NOT

Combine multiple conditions in WHERE clause.

-- AND operator (all conditions must be true)
SELECT column1, column2
FROM table_name
WHERE condition1 AND condition2;

-- OR operator (any condition can be true)
SELECT column1, column2
FROM table_name
WHERE condition1 OR condition2;

-- NOT operator (negates a condition)
SELECT column1, column2
FROM table_name
WHERE NOT condition;

üîç LIKE Operator

Used in WHERE clause to search for a specified pattern.

SELECT column1, column2
FROM table_name
WHERE columnN LIKE pattern;
Pattern examples:
WHERE CustomerName LIKE 'a%' - Starts with "a"
WHERE CustomerName LIKE '%a' - Ends with "a"
WHERE CustomerName LIKE '%or%' - Has "or" in any position
WHERE CustomerName LIKE '_r%' - Has "r" in the second position

‚ûï INSERT INTO

Inserts new records in a table.

-- Specify both column names and values
INSERT INTO table_name (column1, column2, column3)
VALUES (value1, value2, value3);

-- If adding values for all columns
INSERT INTO table_name
VALUES (value1, value2, value3);

‚úèÔ∏è UPDATE

Modifies existing records in a table.

UPDATE table_name
SET column1 = value1, column2 = value2
WHERE condition;
Warning: Be careful when updating records. If you omit the WHERE clause, ALL records will be updated!

üóëÔ∏è DELETE

Deletes existing records from a table.

DELETE FROM table_name WHERE condition;
Warning: Be careful when deleting records. If you omit the WHERE clause, ALL records will be deleted!

üì§ SELECT INTO

Copies data from one table into a new table.

-- Copy all columns into a new table
SELECT *
INTO new_table
FROM old_table
WHERE condition;

-- Copy only some columns
SELECT column1, column2
INTO new_table
FROM old_table
WHERE condition;

üìã IN Operator

Allows specifying multiple values in a WHERE clause.

-- With list of values
SELECT column_name
FROM table_name
WHERE column_name IN (value1, value2, ...);

-- With subquery
SELECT column_name
FROM table_name
WHERE column_name IN (SELECT statement);

‚ÜîÔ∏è BETWEEN Operator

Selects values within a given range (inclusive).

SELECT column_name
FROM table_name
WHERE column_name BETWEEN value1 AND value2;
Example:
SELECT * FROM Products WHERE Price BETWEEN 10 AND 20;

üè∑Ô∏è Aliases

Give a table or column a temporary name.

-- Column alias
SELECT column_name AS alias_name
FROM table_name;

-- Table alias
SELECT column_name
FROM table_name AS alias_name;

üîÑ UNION Operator

Combines the result-set of two or more SELECT statements.

-- UNION (removes duplicates)
SELECT column_name FROM table1
UNION
SELECT column_name FROM table2;

-- UNION ALL (keeps duplicates)
SELECT column_name FROM table1
UNION ALL
SELECT column_name FROM table2;

üìã SQL Clause Summary

SQL Clause Query Example Description
SELECT SELECT * FROM table; Retrieves all or specific columns from a table
SELECT DISTINCT SELECT DISTINCT column FROM table; Retrieves unique values from a column
ORDER BY SELECT column FROM table ORDER BY column; Sorts query results in ascending or descending order

AND, OR, NOT SELECT column FROM table WHERE condition1 AND condition2; Combines multiple conditions in a query
LIKE SELECT column FROM table WHERE column LIKE '%a'; Finds values matching a pattern
INSERT INTO INSERT INTO table (col1, col2) VALUES ('val1', 'val2'); Adds new records to a table
UPDATE UPDATE table SET col1 = 'val1' WHERE condition; Modifies existing records in a table

ALIASES SELECT column1 AS alias1 FROM table; Renames columns or tables temporarily

‚öñÔ∏è DELETE vs TRUNCATE vs DROP

Command Description Rollback? Performance
DELETE Removes specific rows based on condition ‚úÖ Yes (if inside a transaction) Slower, as it logs each row deleted
TRUNCATE Removes all rows but keeps table structure ‚ùå No Faster than DELETE, but no rollback
DROP Deletes the entire table (structure + data) ‚ùå No Fastest, but irreversible
