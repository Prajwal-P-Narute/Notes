Before installing JMeter, you need to set up the required dependencies on your macOS system.

1Ô∏è‚É£ Install Homebrew

Package manager for macOS to install software easily

/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

Verify installation:

If version shows up, Homebrew is ready!

2Ô∏è‚É£ Install Java JDK 17

JMeter requires Java to run

Set Java PATH environment variable

So your system can find Java commands like java.

nano ~/.zprofile

Add this line at the end:

Reload config to apply changes:

source ~/.zprofile

Verify Java version:

java -version

‚úîÔ∏è You should see: openjdk version "17.x.x"

A step-by-step guide to building a reusable performance testing framework

üßµ Thread Group ‚Äì Virtual User Setup

What is Thread Group?

  • Defines number of virtual users
  • Controls test execution loops and ramp-up time

Important Fields:

  • Number of Threads (Users)
  • Ramp-Up Period (seconds)
  • Loop Count

JMeter Components

üß© CSV Data Set Config ‚Äì Parameterization

Purpose:

  • Load dynamic test data (usernames, endpoints, etc.)

Key Fields:

  • Filename: Path to .csv file
  • Variable Names: Headers used as variables
  • Recycle on EOF: Set to false for one-time use
  • Stop Thread on EOF: Ends test if data ends

üõ† User Defined Variables ‚Äì Central Config

Usage:

  • Declare global variables for base URL, environment, tokens

Example:

baseURL = https://api.test.com
env = QA1
Call Variable: ${baseURL}

üì° HTTP Request Samplers (POST & GET)

Dynamic Naming:

POST_${SCENARIOS} / GET_${SCENARIOS}

Usage:

  • Test multiple APIs using one sampler
  • ${SCENARIOS} comes from CSV or predefined variable

üß† If Controller ‚Äì Conditional Execution

Purpose:

  • Execute logic only if a condition is met

Syntax:

${__javaScript(${token} != "")}

üíª JSR223 PreProcessor ‚Äì Scripting Power

Used For:

  • Preparing request data
  • Setting runtime variables
  • Logging

Sample Code (Groovy):

def user = vars.get("username")
log.info("Running test for: " + user)

üîç JSON Extractor ‚Äì Response Parsing

Purpose:

  • Extract values like authToken from API response

Example:

  • Variable Name: token
  • JSON Path: $.data.token
