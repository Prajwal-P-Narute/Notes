Wire Mocking Notes

A comprehensive guide to understanding wire mocking in software testing

What is Wire Mocking?
  • Faking a conversation between your software and an external website/service
  • Intercepts outgoing requests and returns fake, pre-written responses
  • Think of it as a "stunt double" for the internet
  • Allows testing without depending on real external services
  • Essential for testing edge cases and error scenarios

How It Works
  • Setup a "mock server" on your computer
  • Program it with specific responses to requests
  • Redirect your app to the mock server
  • Test your app with the fake responses
  • Simulate different HTTP status codes
  • Test timeout scenarios by delaying responses
  • Verify that your app makes correct requests
Popular Tools
  • WireMock: The most popular standalone tool
  • Nock: For JavaScript/Node.js apps
  • MockServer: Another powerful alternative
  • Postman Mock Server: Easy setup with Postman
  • Jest: For JavaScript testing with mocking capabilities
  • Mockito: Popular for Java applications
  • Pytest: For Python with mocking libraries
Example: WireMock Setup
// Example of setting up a WireMock stub
curl -X POST http://localhost:8080/__admin/mappings/new \
-d '{
  "request": {
    "method": "GET",
    "url": "/user/1"
  },
  "response": {
    "status": 200,
    "jsonBody": {
      "id": 1,
      "name": "Walter White",
      "occupation": "Chemist"
    },
    "headers": {
      "Content-Type": "application/json"
    }

Example: Nock Implementation
// Example using Nock for API mocking in JavaScript
const nock = require('nock');

// Mock a GET request to a user endpoint
nock('https://api.example.com')
  .get('/user/123')
  .reply(200, {
    id: 123,
    username: 'john_doe',
    email: 'john@example.com'
  });

// Now any request to https://api.example.com/user/123
// will return this mock response
