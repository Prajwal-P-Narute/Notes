Quick, copyable commands and short explanations so you can reinstall Jenkins cleanly on a MacBook.

Install Jenkins (recommended: Homebrew)

Steps using Homebrew (fast, integrates with macOS services):

Install Homebrew (if needed):

# Install Homebrew
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# Verify brew
brew --version

Install Jenkins LTS and start as a service:

# Install Jenkins LTS
brew install jenkins-lts

# Start Jenkins as user service (starts at login)
brew services start jenkins-lts

# Stop when needed
brew services stop jenkins-lts

Alternative: run Jenkins from the jenkins.war file (Java required):

# Download jenkins.war from https://www.jenkins.io/download/
# Run it (requires Java 11+)
java -jar ~/Downloads/jenkins.war
Open http://localhost:8080 in your browser after starting Jenkins.

Uninstall Jenkins (clean removal)

Remove Jenkins depending on how it was installed.

Homebrew installation

# Stop the service
brew services stop jenkins-lts

# Uninstall package
brew uninstall jenkins-lts

# Optional: clean up Homebrew
brew cleanup

Remove Jenkins Home (this deletes jobs, plugins, users — irreversible)

# Remove Jenkins home directory
rm -rf /Users/$(whoami)/.jenkins

.war-based installation

# If you ran a jenkins.war manually, delete the war file and the Jenkins home
rm -f ~/Downloads/jenkins.war
rm -rf /Users/$(whoami)/.jenkins

Backup configurations before deleting

# Backup before deleting
cp -r /Users/$(whoami)/.jenkins ~/jenkins-backup-$(date +%F)

Find the initial admin password

After a fresh install, Jenkins creates an initial admin password used to unlock the setup wizard.

# Read the initial admin password
cat /Users/$(whoami)/.jenkins/secrets/initialAdminPassword
If the file is missing, either Jenkins didn't finish setup or the home directory is different. Some points to check:
  • If you ran java -jar jenkins.war, Jenkins home defaults to ~/.jenkins unless overridden by the JENKINS_HOME environment variable.
  • To find where Jenkins is using as home (when running as a service), check the brew service plist or the process command line.
# See JENKINS_HOME in running process (example)
ps aux | grep jenkins
# or inspect the plist used by brew services
cat ~/Library/LaunchAgents/homebrew.mxcl.jenkins-lts.plist

Basic setup steps after installation

  1. Open http://localhost:8080 and paste the initial admin password to unlock.
  2. Choose Install suggested plugins for a typical setup.
  3. Create the first admin user (or skip and use the initial password).
  4. Configure system settings: Manage Jenkins → Configure System (set email, JDK locations, build tools).
  5. Set up credentials: Credentials → System → Global credentials (add SSH keys, tokens, etc.).
  6. Create your first job: New Item → Freestyle project / Pipeline.
  7. Install common plugins (if not already): Pipeline, Git, GitHub, GitLab, Matrix Authorization Strategy, Blue Ocean (optional).

Useful commands & notes

# Check Jenkins logs (Homebrew service)
# Logs typically in ~/Library/Logs/Homebrew/jenkins-lts
ls -la ~/Library/Logs/Homebrew/jenkins-lts

# Check running service
brew services list

# If using jenkins.war – logs appear in console where you started it
If you want Jenkins to run on a different port, set the --httpPort flag when running the war, or configure the launch plist for the brew service. 
# Example: start war on port 9090
java -jar ~/Downloads/jenkins.war --httpPort=9090
