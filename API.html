API Testing Notes

A comprehensive guide to API testing concepts, tools, best practices, and interview preparation

What is API Testing?

API (Application Programming Interface) testing is a type of software testing that validates:

  • Endpoints, request/response
  • Business logic and data handling
  • Security and performance aspects
  • Focuses on backend testing (without UI)

Typically done using tools like Postman, REST Assured, Pytest, SoapUI, Karate, JMeter, etc.

Why API Testing?
  • Early defect detection (before UI is ready)
  • Ensures functionality, reliability, performance, and security
  • Saves time in regression compared to UI automation
  • Validates integration between services
  • Language-agnostic testing approach
  • More stable than UI tests (less prone to changes)
Types of API
  • REST API (JSON/XML, HTTP methods) - Most common
  • SOAP API (XML, WSDL-based) - More structured
  • GraphQL API (query-based) - Efficient data retrieval
  • gRPC API (high-performance, protocol buffers) - Microservices
  • WebSocket APIs - Real-time communication

API Testing Checklist
  • Verify status codes (200, 201, 400, 401, 403, 404, 500, etc.)
  • Validate response body (correct fields, values, data types)
  • Check headers (Content-Type, Authorization, CORS, etc.)
  • Authentication (Basic Auth, OAuth, JWT, API Keys)
  • Error handling (invalid input, missing params)
  • Data integrity (response matches DB / expected schema)
  • Performance (response time, load testing)
  • Security (SQL injection, XSS, token expiration)

Best Practices
  • Always test with valid and invalid inputs
  • Use environment variables (dev, QA, prod)
  • Implement data-driven testing
  • Mock APIs if backend is not ready
  • Validate both request and response payloads
  • Automate regression cases
  • Maintain API documentation
  • Test for edge cases and boundary values
  • Include performance testing in test strategy
Sample API Test Cases
  • Verify GET /users returns 200 OK with correct data
  • Verify POST /login with valid creds returns token
  • Verify POST /login with invalid creds returns 401 Unauthorized
  • Verify PUT /user/{id} updates user details
  • Verify DELETE /user/{id} removes user and returns 204
  • Verify response schema matches API documentation
  • Verify API response time < 2 seconds
  • Verify pagination, sorting, filtering
  • Verify rate limiting works correctly
Interview Tips & Questions
Q1: Difference between API Testing and UI Testing?

Answer: API Testing focuses on backend validation â€” request/response, status codes, business logic, DB validation, and performance. It does not require a user interface. UI Testing focuses on frontend functionality, visual elements, user interactions, and end-to-end workflows.

Q2: REST vs SOAP differences?

Answer: REST uses HTTP methods (GET, POST, PUT, DELETE), is lightweight (usually JSON), faster, and more flexible. SOAP uses XML with strict standards (WSDL), is heavier, requires more bandwidth, and has built-in security and ACID compliance.

Q3: How do you validate API responses with DB?

Answer: After API execution, connect to the database using JDBC (Java) or similar libraries. Run SQL queries to fetch data and compare API response JSON fields with DB values to ensure data integrity.
