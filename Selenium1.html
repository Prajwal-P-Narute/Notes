Automation Testing

Using automated tools and scripts to test application functionality instead of manual testing.

Disadvantages of Manual Testing

  • Longer project duration
  • Requires more human effort
  • Slow regression testing
  • Difficult cross-browser testing

Advantages of Automation

  • Reusable test scripts
  • Faster cross-browser testing
  • Reduces project time & cost
  • Less human effort
  • Reliable & efficient

Advantages of Selenium

  • Open-source (widely used by 80-90% of companies)
  • Supports multiple programming languages
  • Enables cross-browser testing

Disadvantages of Selenium

  • Only supports web applications (not desktop/mobile apps)
  • Cannot automate CAPTCHA or OTP
  • Does not support file uploads
  • Cannot handle barcode scanning
  • Does not automate ad hoc test cases

Selenium Architecture (Client-Server Model)

Key Components

  • Test Script â€“ Written in Java, Python, or C# to define actions.
  • WebDriver â€“ Acts as a bridge between script & browser.
  • Browser Driver â€“ Communicates with the browser (e.g., ChromeDriver).

How Selenium Works (Simple Flow)

  1. Test Script sends commands (e.g., click a button).
  2. WebDriver converts them into browser-understandable actions.
  3. Browser Driver sends instructions to the actual browser.
  4. Browser executes actions and sends results back to WebDriver.

Advantages of Selenium Architecture

  • âœ… Cross-Browser Testing â€“ Works on Chrome, Firefox, Safari, etc.
  • âœ… Multi-Language Support â€“ Java, Python, C#, etc.
  • âœ… Parallel Testing â€“ Runs tests on multiple browsers/machines (via Selenium Grid).

Difference between get() and navigate() methods

Feature get() Method navigate() Method
Purpose Launches a new URL. Launches a URL with additional navigation controls.
History Handling Does not maintain history. Maintains browser history (can go back, forward, or refresh).
Page Load Waits for the page to fully load. Does not wait for full page load.
Syntax driver.get("https://example.com"); driver.navigate().to("https://example.com");
Navigation Features Only loads the page. Supports back, forward, refresh.
Code Simplicity Easier to write. Slightly longer syntax.
Usage Order Used before navigate(). Used after launching a URL.
// Example of navigate() methods
driver.navigate().to("https://www.example.com");
driver.navigate().back(); // Go to previous page
driver.navigate().forward(); // Move forward in history
driver.navigate().refresh(); // Refresh the page

Locators in Selenium

What are Locators?

Locators help identify and interact with web elements on a web page during automation.

Types of Locators

  • Attribute-Based Locators â€“ id, name, class name, accessibility id, text()
  • Tag Name â€“ Uses the HTML tag itself (e.g., <button>, <input>, etc.).
  • XPath â€“ Advanced method to locate elements.
  • CSS Selector â€“ Uses CSS patterns to find elements.

XPath in Selenium

ðŸ”¹ XPath (XML Path Language) is a flexible way to locate elements based on structure and attributes.

Types of XPath

  • âœ… XPath by Attribute â†’ //tag[@attribute='value']
  • âœ… XPath by Text â†’ //tag[text()='text value']
  • âœ… XPath by Contains â†’ //tag[contains(@attribute, 'partial value')]
  • âœ… Absolute XPath â†’ Uses the full path from the root (/html/body/...) but not recommended due to maintenance issues.
  • âœ… Relative XPath â†’ Shorter and flexible path (//parent//child), preferred for automation.
  • âœ… XPath by Index â†’ Used when multiple elements share the same attributes ((//tag[@attribute='value'])[index]).

Special XPath Keywords

  • âœ” Parent â†’ Finds the immediate parent of an element.
    //input[@type='text']/parent::div
  • âœ” Ancestor â†’ Finds the grandparent or higher elements.
    //input[@id='pass']/ancestor::div
  • âœ” Sibling â†’ Finds elements at the same level.
    //input[@id='pass']/following-sibling::div (Next element)
    //input[@id='pass']/preceding-sibling::div (Previous element)
  • âœ” AND Condition â†’ Both conditions must be true.
    //input[@name='email' and @id='username']
  • âœ” OR Condition â†’ At least one condition must be true.
    //input[@id='Gmail' or @name='userid']

Common Exceptions in Selenium & Solutions

Selenium exceptions occur when an error or unexpected condition arises while executing a WebDriver script.

Exception Reason Solution
NoSuchElementException WebDriver cannot find the element. Verify locator, use implicit or explicit waits
TimeoutException Operation takes too long. Increase wait time, use explicit waits
ElementNotInteractableException Element exists but is hidden or disabled. Ensure element is visible, use WebDriverWait
NoSuchWindowException Trying to switch to a closed window. Ensure target window is open
NoSuchFrameException Switching to non-existent frame. Verify frame locator, ensure frame is loaded
StaleElementReferenceException Element is no longer attached to DOM. Relocate element, use explicit waits
InvalidElementStateException Trying to modify read-only element. Verify element's state before interacting
WebDriverException General error (browser crash, driver mismatch). Ensure browser and WebDriver versions match
MoveTargetOutOfBoundsException Mouse moved outside visible screen. Verify element location
SessionNotCreatedException WebDriver fails to create a session. Ensure browser and WebDriver versions match
ElementClickInterceptedException Another element is blocking the target. Scroll to element, wait until clickable
FileNotFoundException File missing or path incorrect. Use correct file path
NullPointerException Using uninitialized object reference. Ensure objects are initialized before use

Best Practices to Handle Exceptions

  • âœ… Use Try-Catch Blocks to prevent crashes.
  • âœ… Implement Explicit Waits for dynamic elements.
  • âœ… Verify Locators (XPath, CSS Selectors) before execution.

Dropdown Handling

To handle dropdown functionality in Selenium:

  1. Find the dropdown element using XPath and store it in a WebElement reference
  2. Create a Select class object passing the WebElement reference
  3. Use Select class methods to interact with the dropdown
// Example of dropdown handling
WebElement ref = driver.findElement(By.xpath("xpath_expression"));
Select ref1 = new Select(ref);

// Various dropdown operations
ref1.selectByIndex(index);
ref1.selectByValue("value");
ref1.selectByVisibleText("text");
ref1.deselectAll();
ref1.getFirstSelectedOption();
ref1.getOptions();

Mouse Actions

Selenium's Actions class enables advanced mouse interactions:

Action Method Usage
Hover (Mouse Over) moveToElement(element) Hover over an element
Right Click contextClick(element) Opens right-click menu
Double Click doubleClick(element) Double-clicks an element
Drag and Drop dragAndDrop(source, target) Moves element from source to target
Click and Hold clickAndHold(element) Clicks and holds without releasing
Release release() Releases the held click
// Example of mouse actions
Actions actions = new Actions(driver);
WebElement element = driver.findElement(By.id("menu"));

actions.moveToElement(element).perform(); // Hover
actions.contextClick(element).perform(); // Right Click
actions.doubleClick(element).perform(); // Double Click

Key Points:

  • Always use perform() to execute the action
  • For complex interactions, chain multiple actions using build().perform()
  • moveToElement() is useful for hover menus
  • dragAndDrop() handles draggable elements

Waits in Selenium

Synchronization ensures test scripts run at the right pace with the application:

1. Unconditional Synchronization (Static Wait)

Thread.sleep(5000); // Wait for 5 seconds

Not recommended for production tests as it's inflexible

2. Conditional Synchronization (Dynamic Waits)

Implicit Wait

driver.manage().timeouts().implicitlyWait(Duration.ofSeconds(10));

Applied globally to all elements. Activates when NoSuchElementException occurs.

Explicit Wait

WebDriverWait wait = new WebDriverWait(driver, Duration.ofSeconds(10));
wait.until(ExpectedConditions.visibilityOfElementLocated(By.xpath("xpath")));

Applied to specific elements. Most preferred wait mechanism.

Fluent Wait

Wait<WebDriver> wait = new FluentWait<WebDriver>(driver)
    .withTimeout(Duration.ofSeconds(10))
    .pollingEvery(Duration.ofSeconds(2))
    .ignoring(NoSuchElementException.class);

Most flexible wait with configurable polling and exception ignoring.

IFrame Handling

  • IFrame is the external webpage which is implemented within main HTML page.
  • Main purpose of using IFrame is to implement another external sources like videos, map, webpages with different technology like Flutter, Angular and advertise etc. into main HTML page document.
  • We cannot handle IFrame directly in Selenium, to handle IFrame we have to switch focus on IFrame first.
  • To handle the IFrame there is IFrame tag is present within HTML code.
<html>
    <iframe></iframe>
</html>

To switch the focus from main window to frame window, we have switchTo() method which returns the TargetLocator and from TargetLocator call frame() method which takes three types of arguments:

  1. Index
  2. Name of IFrame
  3. WebElement
// Switching to IFrame by index
driver.switchTo().frame(0);

// Switching to IFrame by name
driver.switchTo().frame("Iframe");

// Switching to IFrame by WebElement
driver.switchTo().frame(driver.findElement(By.xpath("")));

After switching the focus on IFrame, then we can easily perform the actions on IFrame window.

To navigate/switch the focus from IFrame window to the immediate frame, we have a parentFrame() method:

driver.switchTo().parentFrame();

To navigate/switch the focus from IFrame to the main page, we have a defaultContent() method:

driver.switchTo().defaultContent();

Process of Logging the Defect in Automation

Logging a defect in automation ensures that the issue is properly documented, tracked, and addressed. Following is the general process of logging defect in automation:

  1. Identification the Defect during execution: Find and recognize the issue during test execution.
  2. Capture the Defect Details: Collect error messages, logs, screenshots and other relevant details.
  3. Create automated report: Generate report with defect description, severity, test steps and environment details.
  4. Automation tool Integration: Connect with a Defect Tracking tool such as Jira, Bugzilla through API integrations.
  5. Notification to Relevant Stakeholders: Notify QA engineer, developer and PM via email through defect tracking tool.
  6. Monitoring and Follow-up: Track the status of defect (open, work in progress, retest, reopen and close).
  7. Documentation and Analysis: Analyze the root cause to prevent similar issue in future.

Broken Links Validation

To validate or handle the broken links on the web page, follow this step-by-step approach:

  1. Visit the web page: Navigate to the page you want to test.
  2. Identify all links on the web page: Use the <a> tag to find all links present on the web page.
  3. Send HTTP request to each link: Create object of HttpURLConnection class to send HTTP request to each link.
  4. Check the response code: If response code is 200, the link is valid. For codes 400+, the link is broken.
  5. Handle exception: Handle checked exceptions like MalformedURLException, IOException using try-catch block or throws keyword.

Test script to validate broken links

// Setup WebDriver
WebDriverManager.chromedriver().setup();
ChromeOptions opt = new ChromeOptions();
opt.addArguments("--remote-allow-origins=*");
WebDriver driver = new ChromeDriver(opt);

// Browser configuration
driver.manage().window().maximize();
driver.manage().deleteAllCookies();
driver.get("https://www.google.com/");

// Perform a search
WebElement search_input = driver.findElement(By.xpath("//textarea[@type='search']"));
search_input.sendKeys("java");
search_input.sendKeys(Keys.ENTER);

// Find all links
List<WebElement> links = driver.findElements(By.xpath("//a"));
System.out.println(links.size());

// Check each link
for (int count = 0; count < links.size(); count++) {
    String url = links.get(count).getAttribute("href");
    System.out.println("URL: " + url);

    // Check if the URL is not null and is an HTTP link
    if (url != null && url.startsWith("http")) {
        HttpURLConnection httpURLConnection = (HttpURLConnection) (new URL(url).openConnection());
        httpURLConnection.setRequestMethod("HEAD");
        httpURLConnection.connect();
        int responseCode = httpURLConnection.getResponseCode();
        System.out.println("response code: " + responseCode);

        if (responseCode >= 400) {
            System.out.println("It is a broken link: " + url);
        } else {
            System.out.println("It is a valid link: " + url);
        }
    }
}
System.out.println("***");
