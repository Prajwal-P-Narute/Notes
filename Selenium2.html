üìÑ Page Object Model (POM)

POM is a design pattern in Selenium used to create separate classes for web pages to improve test case maintainability.

// Example of POM class structure
public class LoginPage {
    // Using @FindBy annotation
    @FindBy(xpath = "//span[text()='Admin']")
    private WebElement admin;

    // Constructor with PageFactory initialization
    public LoginPage(WebDriver driver) {
        PageFactory.initElements(driver, this);
    }

    // Method to interact with the element
    public void clickAdmin() {
        admin.click();
    }
}

Advantages of POM

  • Reduces code duplication
  • Makes test scripts more readable
  • Improves maintainability of test cases

Disadvantages of POM

  • Requires time and effort for setup
  • Requires skilled resources

‚ö° TestNG

TestNG is a testing framework inspired by JUnit with more powerful features like annotations, grouping, parameterization, and assertions.

Key Features of TestNG

  • Execute multiple test cases at once via test suite
  • Generate attractive reports in XML and HTML formats
  • Control test execution flow with annotations
  • Set test priorities and dependencies
  • Group test cases for selective execution
  • Compare actual and expected results with assertions
  • No need for main method

TestNG Annotations

Annotation Purpose
@Test Marks a method as a test case
@BeforeSuite Runs before all tests in the suite
@AfterSuite Runs after all tests in the suite
@BeforeTest Runs before any test method
@AfterTest Runs after all test methods
@BeforeClass Runs before the first test method
@AfterClass Runs after all test methods in the class
@BeforeMethod Runs before each test method
@AfterMethod Runs after each test method
Installing TestNG in Eclipse:

  2. Search for "TestNG for Eclipse"
  3. Install and accept licenses
  4. Add TestNG dependency in pom.xml

üîë TestNG Keywords/Parameters

Keyword Description Example
priority Controls execution order (lower numbers first) @Test(priority = 1)
invocationCount Runs a test multiple times @Test(invocationCount = 5)
dependsOnMethods Specifies test dependencies @Test(dependsOnMethods = {"login"})
enabled Enables or disables tests @Test(enabled = false)
timeOut Fails test if execution exceeds time limit (ms) @Test(timeOut = 5000)
// Example of TestNG keywords usage
public class TestNGExample {
    @Test(priority = 1)
    public void login() {
        System.out.println("Login test");
    }

    @Test(priority = 2, dependsOnMethods = {"login"})
    public void dashboard() {
        System.out.println("Dashboard test");
    }

    @Test(invocationCount = 3, timeOut = 1000)
    public void repeatTest() {
        System.out.println("Repeated test");
    }
}

üè∑Ô∏è TestNG Groups

TestNG groups allow you to organize tests flexibly by defining groups for selective execution.

// Example of TestNG groups
public class GroupTests {
    @Test(groups = {"sanity", "regression"})
    public void test1() {
        System.out.println("Sanity and Regression test");
    }

    @Test(groups = {"sanity"})
    public void test2() {
        System.out.println("Sanity test");
    }

    @Test(groups = {"regression"})
    public void test3() {
        System.out.println("Regression test");
    }
}

<groups>
    <run>
        <include name="sanity" />
        <exclude name="regression" />
    </run>
</groups>
Meta Groups: Groups of groups defined using the <define> tag for better organization of tests across multiple modules and sprints.

üëÇ Listeners in TestNG

Listeners are interfaces that allow customization of test execution behavior for logging, reporting, retrying tests, and taking screenshots.

Types of TestNG Listeners

Listener Purpose
ITestListener Modifies individual test case events
ISuiteListener Modifies TestNG suite events
IReporter Modifies reports and logs
IAnnotationTransformer Modifies TestNG annotations
IInvokedMethodListener Listens to method invocations

Real-World Use Cases of Listeners

  • Logging test execution details
  • Capturing screenshots on failure
  • Automatically retrying failed tests
  • Generating detailed HTML or JSON reports
  • Performing setup or cleanup tasks
// Example of ITestListener implementation
public class TestListener implements ITestListener {
    @Override
    public void onTestFailure(ITestResult result) {
        // Capture screenshot on test failure
        System.out.println("Test failed: " + result.getName());
    }

    @Override
    public void onTestSuccess(ITestResult result) {
        System.out.println("Test passed: " + result.getName());
    }
}

‚úÖ Assertions in TestNG

Assertions compare actual and expected results to determine test case pass/fail status.

Type Description Example
Hard Assert Stops execution immediately on failure Assert.assertEquals(actual, expected);
Soft Assert Continues execution after failure, reports at the end softAssert.assertEquals(actual, expected);
// Hard Assert examples
Assert.assertEquals(actualURL, expectedURL);
Assert.assertNotEquals(actualURL, expectedURL);
Assert.assertTrue(condition);
Assert.assertFalse(condition);
Assert.fail(); // Purposely fail test

// Soft Assert examples
SoftAssert softAssert = new SoftAssert();
softAssert.assertEquals(actualURL, expectedURL);
softAssert.assertNotEquals(actualURL, expectedURL);
softAssert.assertTrue(condition);
softAssert.assertFalse(condition);
softAssert.assertAll(); // Mandatory at the end

‚öôÔ∏è Jenkins

Jenkins is an open-source automation server that helps in continuous integration and delivery of software.

Key Features of Jenkins

  • Deploys code for further testing
  • Automates test execution by scheduling
  • Helps track results and address issues early
  • Deploys code to live environment for end users
  • Automates the entire SDLC using pipeline concept

Jenkins History

  • Originally developed by Sun Microsystem in 2004 as Hudson
  • Renamed to Jenkins when Sun was acquired by Oracle
  • Requires Java 8, 11, or 17
  • Supports Windows, macOS, and Linux

Jenkins Pipeline Stages

  1. Build
  2. Deploy for further testing
  3. Test execution
  4. Release for end user
Creating a Jenkins Pipeline for Maven Project:
  1. Log in to Jenkins and click New Item
  2. Enter job name, select Freestyle or Maven project
  3. Configure General Settings
  4. Set up Source Code Management (Git/SVN)
  5. Set Build Triggers (Poll SCM, GitHub Webhook)
  6. Add Build Step: Invoke top-level Maven targets
  7. Add Post-build Actions (JUnit reports, artifacts)
  8. Configure Email Notifications
  9. Save and Build

üîÑ CI/CD (Continuous Integration/Continuous Deployment)

Continuous Integration

A process where team members integrate their work continuously in a shared repository (GitHub).

  1. DevOps team fetches code from repository
  2. Converts code into build
  3. Performs integration build testing
  4. Sends build to QA for UI testing
  5. Deploys to live environment for end users

Continuous Delivery

Ensures that the build is ready for release. If the build works properly, it's sent to customers who have technical knowledge for installation.

Continuous Deployment

After automation testing, if there are no defects, the build is deployed directly to production without customer involvement (for non-technical customers).

Jenkins Workflow

  1. Attach Git, Maven, Selenium, and Plugins to Jenkins
  2. Developer puts code in GitHub
  3. Jenkins pulls code and sends to Selenium for testing
  4. After testing, Jenkins pulls code and executes
  5. Integrate, deliver, and deploy build using Jenkins
Advantages of Jenkins:
  • Many plugins available with customization options
  • Framework that allows extensive customization
  • Customizable nodes (slaves) for job execution
  • Ability to schedule tasks at regular intervals

üåê Cross-Browser Testing

Cross-browser testing ensures that web applications deliver a consistent experience across different browsers and versions.

// Example of cross-browser testing setup
@Test
@Parameters("browser")
void setup(String browser) throws InterruptedException {
    if(browser.contains("chrome")) {
        WebDriverManager.chromedriver().setup();
        ChromeOptions options = new ChromeOptions();
        options.addArguments("--remote-allow-origins=*");
        driver = new ChromeDriver(options);
    } else if (browser.contains("edge")) {
        WebDriverManager.edgedriver().setup();
        EdgeOptions options = new EdgeOptions();
        options.addArguments("--remote-allow-origins=*");
        driver = new EdgeDriver(options);
    }
    driver.manage().deleteAllCookies();
    driver.navigate().refresh();
}
