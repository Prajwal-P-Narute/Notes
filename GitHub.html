Configure Git Globally

Set Git username

Set Git email

Verify Configuration

git config --list --show-origin

Initialize Repository

Add Files

Commit Changes

Fetch Changes Without Merge

üï∞Ô∏è Rollback & Restore

Undo Unstaged Changes

git restore <file>

Unstage Staged Files

git reset <file>

Undo Last Commit (Before Push)

# Keep changes in working directory
git reset --soft HEAD~1

# Remove changes completely
git reset --hard HEAD~1

Undo Last Commit (After Push)

# Revert commit by creating new one
git revert <commit-hash>
git push origin main

# Force remove commit (Dangerous)
git reset --hard <commit-hash>
git push --force

Restore File from Older Commit

git restore --source=<commit-hash> <file>

‚ùå Rollback a Merge

If merge is not committed

git merge --abort

If merge is already committed

git reset --hard HEAD~1

üåç Remote Repository Concepts

origin

Alias for the remote URL. Used to avoid typing long repo URLs.

upstream

Remote branch that your local branch tracks. Useful for git pull or push operations.

üîÅ Revert Changes from Remote main Branch

Option 1 - Undo Full Merge of abc Branch

# Step 1 - Check commit history
git log --oneline

# Step 2 - Reset to commit before merge
git checkout main
git reset --hard <commit-hash>
git push --force origin main
Warning: This removes all changes from abc

Option 2 - Revert Specific Commits from abc

# Step 1 - Identify commit hashes
git log --oneline

# Step 2 - Revert unwanted commits
git revert <commit-hash>
git push origin main
This keeps history safe and clean

Option 3 - Delete abc Branch (Not Merged)

git push origin --delete abc
