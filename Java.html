Java Programming Guide

Complete reference for Java programming language concepts and fundamentals

Introduction to Java

Java is a widely-used programming language in the IT industry for developing and testing software applications.

  • Discovered by James Gosling at Sun Microsystems in 1995
  • Owned by Oracle since 2009-2010
  • First version: JDK 1.0, Current version: JDK 1.8+
  • Originally called Oak, then Green, finally named Java after Indonesian coffee
  • General-purpose language used for both hardware and software applications
  • Follows WORA (Write Once, Run Anywhere) concept

Advantages of Java

  • Open-source language
  • Object-oriented language for solving real-world problems
  • Write Once, Run Anywhere (WORA)
  • Highly secure compared to other languages
  • Platform independent
  • Portable across systems
  • Automatic garbage collection for better memory management

Uses of Java

  • Desktop application development
  • Web application development
  • Robotics programming
  • Mobile/Android app development
  • Game development
  • Embedded systems

Operators in Java

Arithmetic Operators

  • Addition (+)
  • Subtraction (-)
  • Multiplication (*)
  • Division (/)
  • Modulus (%)
  • Increment (++)
  • Decrement (--)

Logical Operators

  • And (&&)
  • Or (||)
  • Not (!)

Relational Operators

  • Greater than (>)
  • Less than (<)
  • Greater than or equal to (>=)
  • Less than or equal to (<=)
  • Not equal to (!=)

Assignment Operator

  • Assignment (=)

Condition Statements

If Statement

Used when you have one condition and expect a true result

If-Else Statement

Used when you have one condition but expect results in both true and false cases

Nested If

Used when you have multiple conditions that depend on each other

Ladder If (Else-If)

Used when you have multiple conditions that execute parallel to each other

Loops in Java

Loops are used to execute a set of instructions repeatedly until a condition becomes true.

For Loop

Used when the number of iterations is known

for (int i = 0; i < 10; i++) {
  // Code to execute
}

While Loop

Used when the number of iterations is not known

int i = 0;
while (i < 10) {
  // Code to execute
  i++;
}

Do-While Loop

Similar to while loop but executes at least once

int i = 0;
do {
  // Code to execute
  i++;
} while (i < 10);

Variables in Java

Variables are names that provide memory to store information and values.

Local Variables

  • Declared inside the body of a method
  • Destroyed after execution
  • Temporary variables
  • No default value in Java

Global Variables

Declared inside a class but outside the body of methods

Instance Variables (Non-static)

  • Declared without the static keyword
  • Has default values in Java

Static Variables (Class Variables)

  • Declared with the static keyword
  • Memory allocation happens once when the class is loaded
  • Permanent variables

Data Types in Java

Data types represent different sizes and values that can be stored in variables.

Primitive Data Types

Non-Primitive Data Types

  • String - stores sequences of characters
  • Class - user-defined data types
  • Array - stores multiple values of the same type

Strings in Java

Strings are used to store sequences of characters and are immutable in Java.

String Declaration

// Using string literal
String str1 = "Hello World";

// Using new keyword
String str2 = new String("Hello World");

Why Strings are Immutable

  • Security - Strings are used for passwords, URLs, etc.
  • Memory Optimization - Java uses String pool to save memory
  • Thread Safety - Multiple threads can use strings safely
  • Performance - Hash codes are cached for efficiency

Arrays in Java

Arrays are used to store multiple values of the same type in a single variable.

Array Characteristics

  • Homogeneous (same data type)
  • Fixed size (not growable)
  • Elements stored in contiguous memory locations
  • Accessed by index (starts at 0)

Array Declaration

// Declaration and initialization
int[] numbers = {1, 2, 3, 4, 5};

// Declaration then initialization
int[] numbers = new int[5];
numbers[0] = 1;
numbers[1] = 2;

Types of Arrays

  • Single-dimensional arrays - Data stored in rows
  • Multidimensional arrays - Data stored in rows and columns
Note: Accessing an array with an invalid index throws an ArrayIndexOutOfBoundsException.

Methods in Java

Methods are collections of instructions that perform specific tasks.

Main Method

The entry point for JVM to start execution of a Java program

public static void main(String[] args) {
  // Program execution starts here
}
  • public - Access modifier (JVM can access it)
  • static - Can be called without creating an object
  • void - Doesn't return any value
  • main - Method name (predefined)
  • String[] args - Command-line arguments

User-Defined Methods

Methods created by programmers to perform specific tasks

Static Methods

  • Declared with the static keyword
  • Called using class name
  • Cannot be modified (static nature)

Non-Static Methods

  • Declared without the static keyword
  • Called using object reference
  • Can be modified (dynamic nature)
  • More secure than static methods

Constructors in Java

Constructors are special methods used to initialize objects and load non-static methods.

Constructor Rules

  • Name must be same as class name
  • No return type (not even void)
  • Can be overloaded with different parameters

Types of Constructors

Default Constructor

  • Provided by compiler if no constructor is defined
  • No parameters
  • Initializes variables with default values

User-Defined Constructor

  • Created by programmer
  • Can be zero-argument or parameterized

Constructor Calling

Constructors are called using the new keyword:

MyClass obj = new MyClass(); // Calls constructor

Object-Oriented Programming (OOP) Concepts

OOP is used to solve real-world entity problems with six main pillars:

1. Class

A collection of objects that serves as a template for creating objects.

  • Contains variables, methods, constructors, and blocks
  • Does not occupy memory (not a real-world entity)

2. Object

An instance of a class that stores non-static methods and variables.

// Object creation syntax
ClassName objectName = new ClassName();

3. Inheritance

Acquiring properties from one class (superclass) to another (subclass).

  • Uses the extends keyword
  • Promotes code reusability
  • Types: Single, Multilevel, Multiple (not supported), Hierarchical, Hybrid

4. Polymorphism

Allows objects to take multiple forms through method overloading and overriding.

Compile-Time Polymorphism (Method Overloading)

  • Multiple methods with same name but different parameters
  • Determined at compile time

Run-Time Polymorphism (Method Overriding)

  • Subclass provides specific implementation of parent class method
  • Determined at runtime

5. Encapsulation

Wrapping data and methods together in a single unit (class).

  • Data hiding using access modifiers (private, protected)
  • Provides security and control

6. Abstraction

Hiding implementation details and showing only essential features.

  • Achieved through abstract classes and interfaces
  • Focuses on what an object does rather than how it does it

Casting in Java

Converting one type of information into another type of information is called casting.

Primitive Casting

Converting one primitive data type information into another primitive data type.

1. Implicit Casting (Widening)

Converting lower to higher data type information

Order: byte → short → int → float → long → double

Data size increases - also called widening casting

2. Explicit Casting (Narrowing)

Converting higher datatype of information into lower data type

Order: double → long → float → int → short → byte

Data size decreases - also called narrowing casting

3. Boolean Casting

Boolean casting is not supported in Java

Non-Primitive Casting

Converting one class of info into another class

1. Up Casting

Acquiring the properties of sub class into super class

Inheritance is mandatory

// Syntax
SuperClass obj = new SubClass();

// Interface example
InterfaceClass obj = new ImplementationClass();

2. Down Casting

Acquiring the properties of superclass into subclass

Can be performed after upcasting only

// Syntax
SubClass obj = (SubClass) new SuperClass();
// Not supported in Java

Keywords: this and super

Used in non-static methods to access global variables.

this Keyword

Used to access the global variable of same class when global variable name and local variable is same.

public class Example {
  private int value;

  public void setValue(int value) {
    this.value = value; // Using 'this' to access class variable
  }
}

super Keyword

Used to access the global variable of super class into subclass when variable names are same.

Requires extends keyword for inheritance

public class Parent {
  protected int value = 10;
}

public class Child extends Parent {
  private int value = 20;

  public void printValues() {
    System.out.println(super.value); // Prints 10 (parent value)
    System.out.println(this.value); // Prints 20 (child value)
  }
}

Exception Handling

Error vs Exception

Error

  • Problem due to lack of system resources
  • Application cannot catch these problems
  • Unchecked by default
  • Belongs to java.lang.error package
  • Examples: VirtualMachineError, StackOverflowError, OutOfMemoryError, ClassFormatError

Exception

  • Unexpected condition found during program execution
  • Can be handled using throws declaration or try-catch-finally blocks
  • Belongs to java.lang.exception package

Types of Exceptions

1. Checked Exceptions (Compile Time)

  • Generated at compile time
  • Handled or declared using throws keyword
  • Subclass of Exception class
  • Examples: IOException, ClassNotFoundException, InterruptedException

2. Unchecked Exceptions (Runtime)

  • Generated at runtime
  • Handled by try-catch-finally blocks
  • Subclass of RuntimeException class
  • Examples: NullPointerException, ArithmeticException, IndexOutOfBoundsException

Exception Handling Blocks

try {
  // Code that may throw exception
  int result = 10 / 0; // This will throw ArithmeticException
} catch (ArithmeticException e) {
  // Solution code for exception
  System.out.println("Cannot divide by zero: " + e.getMessage());
} finally {
  // Always executes whether exception is found or not
  System.out.println("This block always executes");
}

Collections Framework

The Collections Framework provides classes and interfaces to store and manipulate groups of objects.

Main Interfaces

Collection

  • Root interface of the hierarchy
  • Does not maintain order
  • Accepts duplicates

List

  • Ordered collection (index-based)
  • Allows duplicates
  • Best for retrieval operations
  • Implementations: ArrayList, LinkedList, Vector

Set

  • Does not maintain order
  • Does not allow duplicates
  • Best for insertion operations
  • Implementations: HashSet, LinkedHashSet, TreeSet

Queue

  • Follows FIFO (First-In-First-Out) principle
  • First element added is the first removed
  • Implementations: PriorityQueue, LinkedList (as Queue)
import java.util.LinkedList;
import java.util.Queue;

public class QueueExample {
  public static void main(String[] args) {
    // Create a queue using LinkedList
    Queue<Integer> myQueue = new LinkedList<>();

    // Enqueue (add) elements to the queue
    myQueue.offer(1);
    myQueue.offer(2);
    myQueue.offer(3);

    // Dequeue (remove) elements from the queue
    int firstItem = myQueue.poll();
    int secondItem = myQueue.poll();

    System.out.println(firstItem); // Output: 1
    System.out.println(secondItem); // Output: 2
  }
}

Map

  • Stores key-value pairs
  • Keys are unique
  • Implementations: HashMap, LinkedHashMap, TreeMap, Hashtable

Final, Finally, and Finalize

Final

Keyword used for variables, methods, and classes

  • Final variable: Value cannot be changed (constant)
  • Final method: Cannot be overridden in subclass
  • Final class: Cannot be inherited by another class

Finally

Block used in exception handling with try-catch

  • Always executes whether exception occurs or not
  • Used for cleanup code (closing resources)

Finalize

Method called by garbage collector before object destruction

  • Used for cleanup before garbage collection
  • Can be overridden for custom cleanup logic
  • Not guaranteed to be called immediately

Throw, Throws, and Throwable

Throw

Keyword used to explicitly throw an exception

  • Used within method body
  • Mainly for unchecked exceptions
  • Followed by an instance of exception class
public void checkAge(int age) {
  if (age < 18) {
    throw new ArithmeticException("Access denied - You must be at least 18 years old.");
  }
}

Throws

Keyword used in method declaration to indicate possible exceptions

  • Used with method signature
  • Mainly for checked exceptions
  • Can declare multiple exceptions
public void readFile() throws IOException, FileNotFoundException {
  // Code that may throw these exceptions
}

Throwable

Superclass of all errors and exceptions in Java

  • Root class of exception hierarchy
  • Has two main subclasses: Error and Exception

Collection vs Collections

Collection

  • Collection is an interface which is part of collection framework hierarchy in Java
  • It represents a group of objects known as its elements
  • It is used with List, Set and Queue interfaces for collecting, manipulating and storing data
  • Provides basic operations like add, remove, contains, size, etc.
// Example usage of Collection
import java.util.ArrayList;
import java.util.Collection;

Collection<String> arr = new ArrayList<>();
arr.add("index1-pratik");
arr.add("index2-shubham");

Collections

  • Collections is a utility class which provides static methods
  • Used to operate on collection objects
  • Provides methods for sorting, searching, synchronizing, and other utility operations
  • Cannot be instantiated (private constructor)
// Example usage of Collections utility class
import java.util.ArrayList;
import java.util.Collections;
import java.util.List;

List<String> arr = new ArrayList<>();
arr.add("index1-pratik");
arr.add("index2-shubham");

// Sorting the list
Collections.sort(arr);

Key Differences

Collection (Interface) Collections (Utility Class)
Root interface of the collection hierarchy Utility class with only static methods
Declares methods that all collections will have Cannot be instantiated (private constructor)
Can be instantiated via its subinterfaces (List, Set, Queue) Provides polymorphic algorithms that operate on collections
Used to represent a group of objects as a single unit Contains methods for collection manipulation
Provides basic operations like add, remove, contains, size Provides methods for sorting, searching, synchronizing

Collections Utility Methods

Used to perform operations like sorting, searching, synchronizing, and creating immutable collections

  • sort(List<T> list) - Sorts the specified list into ascending order
  • binarySearch(List<T> list, T key) - Searches the list for the specified object
  • reverse(List<T> list) - Reverses the order of elements in the list
  • shuffle(List<T> list) - Randomly permutes the specified list
  • synchronizedCollection(Collection<T> c) - Returns a synchronized (thread-safe) collection
  • unmodifiableCollection(Collection<T> c) - Returns an unmodifiable view of the collection

Collection vs Stream

Collection

  • Use when you need to store, modify, or access data repeatedly
  • Holds all data in memory
  • Can be modified (add/remove elements)
  • External iteration (using loops)

Stream

  • Use for functional processing and read-only operations
  • Does not store data - processes on demand
  • Cannot be modified (immutable)
  • Internal iteration (no explicit loops)
  • Better for complex transformations and parallel processing

Stream API Methods

Used to perform operations like filtering, mapping, sorting, and collecting elements

  • filter(Predicate<T>) - Filter elements based on a condition
  • map(Function<T, R>) - Transform elements
  • collect(Collectors.toList()) - Collect the result into a list
  • forEach(Consumer<T>) - Perform an action on each element
  • sorted() - Sort elements
  • count() - Count the elements

Stream Example in Automation

// Example: Filtering and processing product names
List<String> filteredSortedNames = driver.findElements(By.className("product-name"))
    .stream()
    .map(WebElement::getText)
    .filter(name -> name.contains("Pro"))
    .map(String::toUpperCase)
    .sorted()
    .collect(Collectors.toList());

// Print the results
filteredSortedNames.forEach(System.out::println);
