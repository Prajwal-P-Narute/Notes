Variables

Variable is a name, and it provide the memory which is used to store the information and variable value.

Variable is always declared in class.

Variable is assigned with data type.

There are two main types of variables:

  1. Local variable
  2. Global variable

Local variable

  • Local variable is nothing, but variable declared inside the body of method.
  • After execution local variable get destroyed.
  • We can say that local variable is a temporary variable.
  • The local variable does not have any default value in Java.

Global variable

  • A variable declared inside the class but outside the body of method is called as Global variable.
  • There are two types:
    1. Non static or Instance variable
    2. Static variable or class variable
Instance variable
  • A variable declared inside the class but outside the body of method without declared as static is called as instance variable.
  • Instance variable has some default value in Java.
Static variable
  • A variable declared inside the class but outside the body of method with static keyword is called as static variable.
  • It is also known as a class variable.
  • Memory allocation for static variable happens only once when the class is loaded in the memory.
  • We can say that this is a permanent variable.

Strings

It is used to store multiple combination of character in "double inverted comma". Ex. special symbol, char, digit, decimal values.

String is a class present in java.lang.package. As its non-primitive data type size is not fix. String extends the object class.

We can declare the string by:

  1. With using new keyword (by using object)
  2. Without using new keyword (by literal)

String is Immutable in nature it means unchangeable or it cannot be modified further but it can override. String stored in heap area. One string is created data cannot be change that it creates new object or it gets stored at new memory location.

Why is String immutable?

  1. Security – Strings are used for passwords, URLs, etc. If they were changeable, it could lead to security risks.
  2. Memory Optimization – Java uses a String pool to save memory. If String were mutable, changing one reference would affect others.
  3. Thread Safety – Since String is immutable, multiple threads can use it safely without conflicts.
  4. Performance Boost – Hash codes of strings are cached, making data structures like HashMap efficient.

JVM is provide the platform for covert the high-level language to low level language means it accept the bytecode as input and provide machine code as output. It is possible due to specific memory present in JVM. This memory divided into 4 areas, in which store the multiple methods, variables. Data type.

  1. Stack area
  2. Static pool area
  3. Heap area
  4. Method area

So, I am going to discuss one by one

1) Stack area

Stack memory are used for execution in which main method, local & reference variables are stored. Stack area is very important for JVM from which start the execution of program. Without writing main method JVM cannot be read program. So, we can say that main method is mandatory. Stack memory is very fast but less in size as compare to heap area.

2) Static pool area

Static pool area is used to stored static methods. It is saved in memory at the compilation time.

3) Heap area

Heap area are used to store non static methods. It is saved in memory at the execution time. So, we can reuse the memory. Whenever we create any object, it is always stored in heap area and reference variable in stack area. Heap area give access to anywhere in application with the help of object & reference variable.

4) Method area

The definition body of static as well as non-static methods are stored in method area of JVM. For binding the static method and definition body by using the reference of class name and it's done at the time of compilation. It's also known as early binding. For binding the non-static method and definition body by using the reference of object and it's done at time of execution.

Additional Information: The JVM memory is divided into several runtime data areas: Method Area, Heap, Stack, PC Registers, and Native Method Stack. The Heap is shared among all threads and stores all objects and arrays. The Stack is thread-specific and stores local variables and partial results.

The main purpose of OOPS is to solve real word entity problems. Real world entity problems mean, how to save memory, how to write complex programs in easy and short way, how to reuse the code.

Class is the collection of objects, in which we can declare multiple things like variable, methods, constructor and blocks. Class does not occupy any memory because it is just template. It is not real-world entity.

3) Inheritance

Inheriting or acquiring the properties from one class into another class is called as inheritance. For inheritance, we need to use 'Extends' keyword after to class name. The class from which properties are inherited is known as super class/parent class/base class. The class in which properties are inherited is called as sub class/child class/derived class. It is helping to reuse and reduce the coding because some another old developer already develops the code. We can reuse this code by using inheritance concept.

Now, I am going to talk about the types of inheritance:

  1. Single level
  2. Multi-level
  3. Multiple
  4. Hierarchical level
  5. Hybrid level
1. Single level

Inheriting or acquiring the properties from only one super class into single subclass, is called as single level inheritance.

2. Multi-level

Inheriting or acquiring the properties from one super class into subclass and this subclass properties inherited into another subclass, is called as Multi level Inheritance.

3. Multiple

In multiple inheritance, the properties of two or more super classes are inherited to one subclass. Multiple inheritance is not supported in java due to diamond ambiguity problem.

Diamond ambiguity problem

In java, there is a one default object class this is known as super most class or user defined superclass. When we are inheriting the properties from two superclass into single subclass by using 'extends' keyword and call the properties of object class into subclass by using object creation. Then at that time compiler get confused to inheriting the properties from both super class and selecting the path from object class. So, this problem is nothing but the ambiguity problem. This problem structure is in diamond shape that why it is called as diamond ambiguity problem.

4. Hierarchical level

In Hierarchical level, inheriting the properties from single superclass into multiple subclasses.

5. Hybrid level

Combination of multiple inheritance types.

4) Polymorphism

Definition: Polymorphism allows objects to take multiple forms, by enabling method overloading and method overriding.

There are two types of polymorphism in java:

  1. Compile time polymorphism
  2. Run time polymorphism
1. Compile time polymorphism

Method declaration and method definition are binder together at the time of program compilation based on argument's creation is called as compile time polymorphism. Method overloading is the example of compile time polymorphism. Method overloading occurs when multiple methods in the same class have the same name but differ in:

  • The number of parameters.
  • The types of parameters.
  • The order of parameters.

Key Points:

  • Happens at compile time (also called Compile-Time Polymorphism).
  • Methods must have the same name but different signatures (method signatures include method name and parameter list).
2. Run time polymorphism

Method declaration and method definition get binder together at the time of program execution based on object creation. Method overriding is the example of run time polymorphism. Method Overriding allows a child class to provide a specific implementation for method that is already defined in its parent class.

The method in the child class must have:

  • The same name.
  • The same parameters.
  • The same return type.

How it Works?

  • Parent Class: Defines a method.
  • Child Class: Rewrites (or overrides) that method to give it a new behaviour.
  • This is used when the child class needs to customize or completely change how the method works.

5) Encapsulation

It's like a capsule that contains data and methods, but you can only access the overall capsule. For example, in POM class we have hide reference variable of web element by using private access modifier and provide access to corresponding method into test classes. For example, in mobile there is mother board is the heart of mobile and we can't access in it but we can access the outside body of mobile.

6) Abstraction

The process of hiding confidential details (coding part) and showing only essential information (functionality of app) to the user this concept is called as abstraction. It is achieved by abstract class or interface class. Ex- whenever any user uses any web application then user can use the functionality of an application, but he can't access the coding part.

Real-Life Use Case in Automation Framework

  • Encapsulation: Organizing locators and actions for each page in a POM.
  • Inheritance: Reusing common setup and utility methods across test classes.
  • Polymorphism: Overloading utility methods or overriding behavior in specific test cases.
  • Abstraction: Defining interfaces for driver initialization, test reporting, or logging.

Method of Overriding in java

Acquiring the methods from superclass into subclass and changing its implementation according to the subclass specification is called as method overriding. Method overriding is done with only non-static method.

Methods hiding

Declaring a static method in superclass and subclass with the same name and same argument is called as method hiding. Method hiding can be done with static method only.

Abstract

Abstract class is incomplete class. Abstract class contain abstract method as well as complete method. When methods declaration is present but method definition is absent is called as abstract method. When both method declaration and definition is present then it is called as complete method. We can create the object of abstract class. For complete abstract method, we need to create concrete class with create their own object.

Concrete class

Concrete class is the complete class. Concrete class is subclass of abstract class. In concrete class, all the abstract method of the abstract class binding with definition body. Ex. Abstract class contain 10 incomplete or abstract method and subclass of that abstract class provide implementation to 5 abstract methods only then this sub class also called is abstract class.

Interface

It is a one of OOPS concept or principle. Interface contain the 100% abstract method before jdk1.8, but after coming the version of jdk1.8 then java introduce the default method. The method declares in interface by static, public and abstract in nature. Variable declares inside the interface by public, static, final in nature. Constructor concepts are not present inside the interface; therefore, we cannot create object of interface.

Implementation class

The class which is complete or provided implementation to all abstract method of interface class with help of "implement" keyword is called as implement class.

Casting in java

Convert one type of information into another type of information is called as casting. There are two types of casting in java:

  1. Primitive casting
  2. Non primitive casting
1) Primitive casting

Converting one primitive data type information into another primitive data type is called as primitive casting. It is divided into three types:

  1. Implicit casting
  2. Explicit casting
  3. Boolean casting
1. Implicit casting/widening casting

Converting lower to higher data type information is called implicit casting. Byte-short-int-float-long-double. Here information size get increase. In this casting, the size of data increases therefore it's also called as widening casting.

2. Explicit casting/narrowing casting

Converting higher datatype of information into lower data type is called as explicit casting that is Double-long-float-int-short-byte. In this casting, the size of data decreased or lost therefore it's also called as narrowing casting.

3. Boolean casting

Boolean casting does not support in java.

2) Non primitive casting

Converting one class of info into another class is called as non-primitive casting. There are two types of non-primitive casting:

  1. Up casting
  2. Down casting
1. Up casting

Acquiring the properties of sub class into super class is known as upcasting. To perform the upcasting, inheritance is mandatory. In a casting, we can perform on those properties which are already inherited from superclass. Syntax of upcasting – SupClass a=new SubClass b (); We can upcast into interface class by using reference of interface class and the object of implementation class IntClass a=new ImpClass b ();

2. Down casting

Acquiring or inheriting the properties of superclass into subclass is called as down casting. Down casting can be performed after the upcasting only. Syntax of down casting. SubClass a= (SubClass) new SupClass (); This is not supported in java.

Keyword

This and super keyword are used in non-static method only to access the global variable.

This

This keyword used to access the global variable of same class when global variable name and local variable is same. [this.variable]

Super

Super keyword is used to access the global variable of super class into subclass, when global variable & sub class variable name is same. [super.variable]. for accessing the variable of superclass, we must use extends keyword.

Exception handling

Error

Error is a problem occurred by programmer due to lack of system resources and application is not able to catch this problem. Unchecked condition is the type of error by default. Error is belonged to java.lang.error package. Example:

  1. Virtual machine error
  2. Stack overflow error
  3. Out of memory error
  4. Class format error

Exception

Exception is nothing but unexpected condition found while executing the program. This type of unexpected condition or problem handle by using add throws declaration and by declaring try, catch finally block. Exception is belonged to java.lang.exception package.

There are two types of exception that is:

  1. Compile time exception or checked exception
  2. Run time exception or unchecked exception
1) Compile time exception or checked exception

The exceptions are generated at the time of compilation, so this is checked exception. Check exception are handled or declare by using throws keyword. Check exception is the subclass of exception class. Example:

  1. I/O exception
  2. ClassNotFound exception
  3. Interrupted exception
2) Run time exception or unchecked exception

The exceptions are generated at the time of runtime, so this is unchecked exception. Unchecked exception is handled by try, catch and finally block.

  1. Try – it contains exception code
  2. Catch- it contains solution code for exception
  3. Finally- finally block is always executed whether the exception is found or not.

Unchecked exceptions are the subclass of runtime exception class. Example:

  1. Null point exception
  2. Arithmetic exception
  3. Index out off bound exception
Exception Handling Example
public class ExceptionExample { public static void main(String[] args) { // Try-catch-finally example try { int result = 10 / 0; // This will throw ArithmeticException System.out.println("Result: " + result); } catch (ArithmeticException e) { System.out.println("Error: Division by zero is not allowed."); } finally { System.out.println("This block always executes."); } // Multiple catch blocks try { int[] numbers = {1, 2, 3}; System.out.println(numbers[5]); // This will throw ArrayIndexOutOfBoundsException } catch (ArrayIndexOutOfBoundsException e) { System.out.println("Error: Array index out of bounds."); } catch (Exception e) { System.out.println("Error: " + e.getMessage()); } // Throws example try { readFile("nonexistent.txt"); } catch (Exception e) { System.out.println("Error: " + e.getMessage()); } } // Method with throws declaration static void readFile(String filename) throws Exception { // Simulate file reading that might throw an exception throw new Exception("File not found: " + filename); } }

Stream API

Collection vs Stream

Use Collection when you need to store, modify, or access data repeatedly. Use Stream for functional processing, read-only operations, or when performing complex transformations.

Stream

Stream API methods used to perform operations like filtering, mapping, sorting, and collecting elements.

When to Use Stream() in Automation:

  • When dealing with lists of elements like product listings, menus, or tables.
  • To avoid loops and make the code more readable and maintainable.

Common Methods Used with Stream():

  1. filter(Predicate) – Filter elements based on a condition.
  2. map(Function) – Transform elements.
  3. collect(Collectors.toList()) – Collect the result into a list.
  4. forEach(Consumer) – Perform an action on each element.
  5. sorted() – Sort elements.
  6. count() – Count the elements.

Example Use Cases in Selenium Automation:

  • Extract product names
  • Filter names containing "Pro"
  • Convert to uppercase
  • Sort alphabetically
  • Collect to a list
Stream API Example
import java.util.*; import java.util.stream.Collectors; public class StreamAPIExample { public static void main(String[] args) { List products = Arrays.asList( "iPhone Pro", "Samsung Galaxy", "Google Pixel", "iPhone 12", "Samsung Note", "Google Nexus" ); // Filter products containing "Pro" List filteredProducts = products.stream() .filter(p -> p.contains("Pro")) .collect(Collectors.toList()); System.out.println("Products with 'Pro': " + filteredProducts); // Convert to uppercase List upperCaseProducts = products.stream() .map(String::toUpperCase) .collect(Collectors.toList()); System.out.println("Uppercase products: " + upperCaseProducts); // Sort alphabetically List sortedProducts = products.stream() .sorted() .collect(Collectors.toList()); System.out.println("Sorted products: " + sortedProducts); // Count elements long count = products.stream() .filter(p -> p.startsWith("iPhone")) .count(); System.out.println("Number of iPhone products: " + count); // ForEach example System.out.println("All products:"); products.stream() .forEach(System.out::println); // More complex example: Filter, transform, and sort List result = products.stream() .filter(p -> p.contains("o")) .map(String::toUpperCase) .sorted() .collect(Collectors.toList()); System.out.println("Filtered, uppercase, sorted: " + result); } }

Additional Information: Java Stream API was introduced in Java 8 to process collections of objects in a functional programming style. A stream is not a data structure; it takes input from Collections, Arrays, or I/O channels. Streams don't change the original data structure; they only provide the result as per the pipelined methods.
